Welcome, friend

Tartuffe is far from the first open source tool for test-doubles. But it can stand in for Mountebank and not have the same pain points. No guarantee that it can cure what ails you, or Mountebank - but it tries to help. (After all, I'm not Moliere, this Tartuffe is meant to help.)

How it works

Tartuffe allows management of imposters in the same fashion as Mountebank. Your test communicates to the server over http using the api to set up stubs, record and replay proxies, and verify mock expectations the same as Mountebank.

Tartuffe focuses on providing the same depth of support for HTTP(s) and TCP, as well as an in-tree implementation of gRPC imposters. API-based management of gRPC imposters is a roadmap feature.

Why tartuffe?

An heretically mechanical replication of Mountebank, wire compatible with the API and serializations, rebuilt in Golang to alleviate some Node-ish pressure points.

Use of agentic code generation to augment an engineer to implement a tool to imitate services for testing the purity of implementation seemed like a perfectly appropriate place for a project to reference the work in which an imposter of piety nearly destroys a family.
