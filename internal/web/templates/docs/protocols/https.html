https

The HTTPS protocol is identical to HTTP except that it uses TLS to encrypt the connection.

Imposter Creation Parameters

All HTTP parameters are supported, plus the following:

Parameter Options Required? Default Description
protocol https Yes N/A  
key PEM-formatted string No A default self-signed key The SSL server private key
cert PEM-formatted string No A default self-signed certificate The SSL server certificate
mutualAuth true or false No false If true, the server will require client certificates

Example

The following example creates a basic HTTPS imposter with default certificates:

POST /imposters HTTP/1.1
Host: localhost:2525
Content-Type: application/json

{
  "port": 4545,
  "protocol": "https",
  "stubs": [{
    "responses": [{
      "is": {
        "statusCode": 200,
        "body": "Secure response!"
      }
    }]
  }]
}

Using Custom Certificates

You can provide your own certificates for more realistic testing:

{
  "port": 4545,
  "protocol": "https",
  "key": "-----BEGIN RSA PRIVATE KEY-----\n...\n-----END RSA PRIVATE KEY-----",
  "cert": "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----",
  "stubs": [...]
}
