tcp

The TCP protocol allows you to stub any TCP-based protocol that tartuffe doesn't have explicit support for. This is useful for binary protocols and custom text protocols.

Imposter Creation Parameters

Parameter Options Required? Default Description
protocol tcp Yes N/A  
port Any valid port number No A randomly assigned port The port to run the imposter on
mode text or binary No text Defines how request/response data is encoded
name Any string No empty string Included in the logs, useful when multiple imposters are set up
recordRequests true or false No false Adds mock verification support
stubs Valid stubs No An empty array The list of stubs

TCP Requests

Field Description Type
requestFrom The client socket address string
data The request data (text or base64-encoded binary) string

TCP Responses

Field Type Default
data string ""

Text Mode Example

{
  "port": 4545,
  "protocol": "tcp",
  "mode": "text",
  "stubs": [{
    "predicates": [{
      "contains": { "data": "HELLO" }
    }],
    "responses": [{
      "is": { "data": "WORLD\n" }
    }]
  }]
}

Binary Mode Example

In binary mode, request and response data is base64-encoded:

{
  "port": 5555,
  "protocol": "tcp",
  "mode": "binary",
  "stubs": [{
    "predicates": [{
      "contains": { "data": "c2F5SGVsbG8=" }
    }],
    "responses": [{
      "is": { "data": "aGVsbG8sIHdvcmxkIQ==" }
    }]
  }]
}

In this example, c2F5SGVsbG8= is the base64 encoding of "sayHello", and aGVsbG8sIHdvcmxkIQ== is the base64 encoding of "hello, world!".
