smtp

The SMTP protocol allows you to create test doubles for email sending functionality. This is useful for testing applications that send emails without actually sending them.

Imposter Creation Parameters

Parameter Options Required? Default Description
protocol smtp Yes N/A  
port Any valid port number No A randomly assigned port The port to run the imposter on
name Any string No empty string Included in the logs
recordRequests true or false No false Adds mock verification support

SMTP Requests

Field Description Type
requestFrom The client socket address string
from The MAIL FROM address string
to The RCPT TO addresses array of strings
subject The email subject string
text The plain text body string
html The HTML body string

Example

POST /imposters HTTP/1.1
Host: localhost:2525
Content-Type: application/json

{
  "port": 2525,
  "protocol": "smtp",
  "recordRequests": true
}

Configure your application to send email to localhost:2525, then retrieve the recorded requests to verify the emails were sent correctly:

GET /imposters/2525 HTTP/1.1
Host: localhost:2525
Accept: application/json

The response will include the recorded email requests in the requests array.
