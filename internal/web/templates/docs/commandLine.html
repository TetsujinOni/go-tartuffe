Configuring tartuffe

The basic command line structure is:

tartuffe [options...]

Command line options

Running tartuffe by itself, without any options, will start up the API on port 2525. It will also spin up this website on http://localhost:2525/, giving you accurate documentation for the version of tartuffe you're running.

Option Description Default
--port 2525 The port to run the main tartuffe server on 2525
--host mbserver.local The hostname to bind the main tartuffe server to all hosts
--datadir .mbdb The root directory for persisting all imposter changes. When used, tartuffe will start all imposters saved in the directory initially and persist all operations to disk in real time, significantly reducing the memory footprint. Without this option, all configuration will be in memory.
--configfile imposters.json If present, tartuffe will load the contents of the specified file. See below for details. N/A
--logfile tartuffe.log The file for tartuffe to store the logs in tartuffe.log
--loglevel debug The logging level, one of debug, info, warn, error info
--nologfile Prevent logging to the filesystem false
--allowInjection Tartuffe supports JavaScript injection for predicates, stub responses, behavior decoration, wait behavior functions and tcp request resolution, but they are disabled by default. Including this parameter will enable them.

Note that allowing injection means that an attacker can run random code on the machine running tartuffe. Please see the security page for tips on securing your system.

 false
--localOnly Only accept requests from localhost. You should ALWAYS do this when running tartuffe with allowInjection directly on your developer machine. false
--ipWhitelist A pipe-delimited string of remote IP addresses to whitelist (local IP addresses will always be allowed). Any request to the primary tartuffe socket or an imposter socket that isn't whitelisted will be dropped. *, representing all IP addresses
--origin A safe origin for CORS requests. Use the flag multiple times to enable multiple origins. false, which disables CORS to prevent CSRF attacks.
--debug Include a matches array with each stub in the body of a GET imposter response for debugging why a particular stub did or did not match a request. false
--apikey myapikey An optional API key. When this is provided, the API will require that the x-api-key header be supplied with a matching key. null
--version Print the version out to the console and exit. N/A
--help Show help for the command N/A

Config files

Tartuffe supports loading imposter configurations from a JSON file at startup using the --configfile option. The file should contain valid JSON matching the imposters contract.

Unlike mountebank, tartuffe currently only supports plain JSON config files without EJS templating. Support for comment stripping is available to allow annotations in your config files.

Example config file:

{
  "imposters": [
    {
      "protocol": "http",
      "port": 4545,
      "stubs": [{
        "responses": [{
          "is": { "body": "Hello, world!" }
        }]
      }]
    }
  ]
}
