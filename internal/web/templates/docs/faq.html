Frequently Asked Questions

General Questions

What is tartuffe?

Tartuffe is an open source tool providing test doubles over the wire. It allows you to create stub servers that respond with canned responses, proxy servers that record and replay traffic, and mock servers that verify expected calls were made.

How is tartuffe different from mountebank?

Tartuffe is a Go implementation compatible with mountebank's API. Key differences:

  • Written in Go instead of Node.js
  • Single binary with no runtime dependencies
  • Lower memory footprint
  • JavaScript injection uses an embedded engine rather than Node.js

Can I use tartuffe as a drop-in replacement for mountebank?

In most cases, yes. Tartuffe aims to be wire-compatible with mountebank's REST API. However, some advanced features (particularly around JavaScript injection and custom formatters) may have subtle differences.

Configuration

How do I start tartuffe?

Simply run tartuffe from the command line. By default, it listens on port 2525. See the command line page for all options.

How do I enable JavaScript injection?

Start tartuffe with the --allowInjection flag. Be sure to read the security page first.

How do I persist imposters between restarts?

Use the --datadir option to specify a directory for persistent storage, or use the --configfile option to load imposters from a JSON file at startup.

Troubleshooting

Why isn't my predicate matching?

Start tartuffe with --debug to enable the matches array in GET /imposters/:port responses. This shows exactly which predicates matched or didn't match for each request.

Why am I getting "port in use" errors?

Each imposter requires a unique port. Make sure you're not trying to create multiple imposters on the same port, and that no other process is using the port.

How do I see the requests made to an imposter?

Create the imposter with "recordRequests": true, then GET the imposter to see the requests array. See the mock verification page for details.
