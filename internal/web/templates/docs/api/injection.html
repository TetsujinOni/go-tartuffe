Injection

Injection allows you to extend tartuffe's functionality using JavaScript. It provides ultimate flexibility for complex testing scenarios that can't be handled with static stub configurations.

Security Warning: Injection is disabled by default. You must start tartuffe with the --allowInjection flag to enable it. See the security page for important security considerations.

Response Injection

You can inject a JavaScript function to dynamically generate responses:

{
  "port": 4545,
  "protocol": "http",
  "stubs": [{
    "responses": [{
      "inject": "function(request) { return { body: 'Hello, ' + request.query.name }; }"
    }]
  }]
}

The function receives the request object and must return a response object.

Predicate Injection

You can inject a JavaScript function to determine if a predicate matches:

{
  "port": 4545,
  "protocol": "http",
  "stubs": [{
    "predicates": [{
      "inject": "function(request) { return request.query.allowed === 'true'; }"
    }],
    "responses": [{
      "is": { "body": "Allowed!" }
    }]
  }]
}

The function receives the request object and must return a boolean.

Available Objects

Injected functions have access to:

Object Description
request The incoming request object (protocol-specific fields)
state A shared state object persisted across all invocations on the imposter
logger A logging object for debugging

Note: Tartuffe's JavaScript injection uses an embedded JavaScript engine. Some Node.js specific APIs may not be available.
