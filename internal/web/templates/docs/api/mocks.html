Mock Verification

While stubs return canned responses, mocks also record the requests made to them for later verification. This allows you to assert that your application made the expected calls to external services.

Enabling Request Recording

To enable mock verification, set recordRequests to true when creating an imposter:

{
  "port": 4545,
  "protocol": "http",
  "recordRequests": true,
  "stubs": [{
    "responses": [{
      "is": { "body": "OK" }
    }]
  }]
}

Retrieving Recorded Requests

After your tests have run, you can retrieve the recorded requests by making a GET request to the imposter:

GET /imposters/4545 HTTP/1.1
Host: localhost:2525
Accept: application/json

The response will include a requests array containing all recorded requests:

{
  "protocol": "http",
  "port": 4545,
  "numberOfRequests": 2,
  "recordRequests": true,
  "requests": [
    {
      "requestFrom": "127.0.0.1:60523",
      "method": "POST",
      "path": "/api/users",
      "query": {},
      "headers": {
        "Content-Type": "application/json"
      },
      "body": "{\"name\": \"John\"}",
      "timestamp": "2024-01-09T02:30:31.022Z"
    }
  ],
  "stubs": [...]
}

Clearing Recorded Requests

To clear recorded requests without deleting the imposter, use:

DELETE /imposters/4545/savedRequests HTTP/1.1
Host: localhost:2525

Note: Recording requests represents a memory leak for any long-running tartuffe process, as requests are never automatically forgotten.
