Faults

Faults allow you to test how your application handles connection-level errors. Instead of returning a normal response, tartuffe can simulate various network faults.

Fault Types

Fault Description
CONNECTION_RESET_BY_PEER Resets the TCP connection, simulating a connection reset by the server.
RANDOM_DATA_THEN_CLOSE Sends random binary data and then closes the connection.

Example

The following example simulates a connection reset:

{
  "port": 4545,
  "protocol": "http",
  "stubs": [{
    "predicates": [{
      "equals": { "path": "/fault" }
    }],
    "responses": [{
      "fault": "CONNECTION_RESET_BY_PEER"
    }]
  }]
}

When a request is made to /fault, the connection will be reset instead of receiving a normal HTTP response.
