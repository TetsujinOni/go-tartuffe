Errors

When something goes wrong, tartuffe returns a JSON error response with diagnostic information to help you troubleshoot the problem.

Error Response Format

All error responses follow this format:

{
  "errors": [{
    "code": "error code",
    "message": "Human-readable error message",
    "source": "Optional source reference"
  }]
}

Common Error Codes

Code HTTP Status Description
bad data 400 The request body contains invalid JSON or missing required fields
invalid injection 400 JavaScript injection is attempted but --allowInjection is not enabled
resource conflict 409 The requested port is already in use by another imposter
no such resource 404 The requested imposter does not exist
invalid protocol 400 The specified protocol is not supported

Example Error Response

If you try to create an imposter on a port that's already in use:

HTTP/1.1 409 Conflict
Content-Type: application/json

{
  "errors": [{
    "code": "resource conflict",
    "message": "Port 4545 is already in use",
    "source": {"port": 4545}
  }]
}
