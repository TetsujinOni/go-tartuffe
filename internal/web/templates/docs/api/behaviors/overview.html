Behaviors

Behaviors are a way of post-processing the response returned by a stub. They provide additional functionality that allows you to modify responses in powerful ways.

The following behaviors are supported:

Behavior Description
wait Adds latency to a response by waiting a specified number of milliseconds before sending.
repeat Repeats a response a specified number of times before moving to the next response.
copy Copies one or more values from the request into the response.
lookup Looks up a value from an external data source and replaces values in the response.
decorate Post-processes the response using JavaScript injection.
shellTransform Post-processes the response by piping it through a shell command.

Multiple behaviors can be chained together. They execute in the order specified in the behaviors array.

Example

The following example shows a stub with a wait behavior that adds 500ms of latency:

{
  "port": 4545,
  "protocol": "http",
  "stubs": [{
    "responses": [{
      "is": {
        "body": "Hello, world!"
      },
      "behaviors": [
        { "wait": 500 }
      ]
    }]
  }]
}
